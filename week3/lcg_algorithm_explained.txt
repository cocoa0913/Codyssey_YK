================================================================================
LCG (Linear Congruential Generator) — 선형 합동 생성기
================================================================================

--------------------------------------------------------------------------------
1. 개요
--------------------------------------------------------------------------------

컴퓨터는 진정한 난수(true random number)를 스스로 만들어낼 수 없다.
대신 수학 공식으로 "난수처럼 보이는" 수열을 생성하는데, 이를
의사난수(pseudo-random number)라고 부른다.

LCG는 의사난수 생성 알고리즘 중 가장 오래되고 구조가 단순한 방식으로,
1948년 D.H. Lehmer가 처음 제안했다.


--------------------------------------------------------------------------------
2. 핵심 점화식 (Recurrence Relation)
--------------------------------------------------------------------------------

    X(n+1) = ( a * X(n) + c ) mod m

    변수 설명:
    - X(n)   : 현재 시드(seed) — 이전 단계에서 계산된 값
    - X(n+1) : 다음 시드 — 이번 단계에서 새로 생성되는 값
    - a      : 곱셈 계수 (multiplier)
    - c      : 덧셈 계수 (increment)
    - m      : 모듈러스 (modulus) — 출력값의 범위를 0 ~ m-1 로 제한

이 공식을 반복 적용하면 0 ~ m-1 사이의 수가 계속 나열되며,
적절한 파라미터를 고르면 m개의 서로 다른 값이 골고루 나타난다.


--------------------------------------------------------------------------------
3. 모듈러 연산(mod)이란?
--------------------------------------------------------------------------------

mod는 나눗셈의 나머지를 구하는 연산이다.

    예)  17 mod 5  =  2   (17 ÷ 5 = 3 ... 나머지 2)
         20 mod 5  =  0   (20 ÷ 5 = 4 ... 나머지 0)

m = 2^32 = 4,294,967,296 으로 설정하면 결과값은 항상 0 ~ 2^32-1 범위에 묶인다.

Python 코드에서는 mod 대신 비트 AND 연산 ( & 0xFFFFFFFF ) 을 사용한다.
    0xFFFFFFFF = 2^32 - 1 = 4,294,967,295 (이진수로 32개의 1)
    어떤 수에 & 0xFFFFFFFF 를 적용하면 하위 32비트만 남으므로 mod 2^32 와 동일하다.
    비트 연산이 나머지 연산보다 CPU에서 훨씬 빠르기 때문에 이 방식을 자주 쓴다.


--------------------------------------------------------------------------------
4. 이 코드에서 사용하는 파라미터 (Numerical Recipes 권장값)
--------------------------------------------------------------------------------

    a = 1,664,525
    c = 1,013,904,223
    m = 2^32 = 4,294,967,296

이 값들은 수치 해석 교재 "Numerical Recipes"에서 검증된 파라미터로,
통계적 균일성이 좋은 것으로 알려져 있다.


--------------------------------------------------------------------------------
5. 동작 예시 (직접 계산해보기)
--------------------------------------------------------------------------------

초기 시드: X(0) = 12345

1단계:
    X(1) = (1,664,525 × 12,345 + 1,013,904,223) mod 2^32
         = (20,548,706,125 + 1,013,904,223) mod 4,294,967,296
         = 21,562,610,348 mod 4,294,967,296
         = 678,708,460

    → 0~1 사이 값으로 변환: 678,708,460 / 4,294,967,295 ≈ 0.1580

2단계:
    X(2) = (1,664,525 × 678,708,460 + 1,013,904,223) mod 2^32
         = ...
         ≈ 다른 0~1 사이 값

이 값을 _uniform(low, high) 함수에 넘기면 원하는 범위의 난수로 변환된다.
    예) 내부 온도 범위 (18, 30) → 18 + (30 - 18) × 0.1580 ≈ 19.90


--------------------------------------------------------------------------------
6. 장점과 단점
--------------------------------------------------------------------------------

[장점]
    - 구현이 매우 단순하다 (곱셈, 덧셈, mod 세 연산이 전부)
    - 실행 속도가 매우 빠르다
    - 시드가 같으면 항상 같은 수열이 나와 재현(reproducibility)이 가능하다
      → 디버깅·시뮬레이션에서 유용

[단점]
    - 주기(period)가 유한하다: 최대 m개의 값이 나오면 수열이 반복된다
    - 수열이 예측 가능하다: 연속된 값 몇 개를 알면 파라미터를 역산할 수 있다
    - 따라서 암호화·보안 목적에는 절대 사용하면 안 된다


--------------------------------------------------------------------------------
7. 어디에 쓰이나?
--------------------------------------------------------------------------------

    적합한 사용처:
    - 게임의 맵·아이템 생성 (씨앗 기반 절차적 생성)
    - 물리/공학 시뮬레이션의 더미 데이터
    - 소프트웨어 테스트용 입력 데이터 생성
    - 이 코드처럼 센서 시뮬레이터의 가짜 측정값 생성

    부적합한 사용처:
    - 비밀번호 생성, 암호화 키 생성
    - 보안 토큰, 세션 ID 생성
    (이런 경우에는 os.urandom() 또는 secrets 모듈을 사용할 것)
